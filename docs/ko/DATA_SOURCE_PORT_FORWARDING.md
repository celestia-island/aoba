# 포트 포워딩 (투명 포트 포워딩 모드)

## 개요

포트 포워딩 모드를 사용하면 TUI 내에서 한 포트에서 다른 포트로 Modbus 데이터를 투명하게 전달할 수 있습니다. 이 기능은 다음과 같은 고급 사용 사례를 가능하게 합니다:

1. **데이터 전달**: 한 포트의 슬레이브 국번을 다른 포트의 마스터 국번으로 변환
2. **데이터 복제**: 마스터 국번 데이터를 모니터링 또는 테스트를 위해 여러 포트에 복제
3. **프로토콜 브릿지**: 서로 다른 물리적 포트 또는 가상 연결 간의 데이터 브릿지

## 포트 포워딩 사용 시기

- **다중 포트 시나리오**: 여러 시리얼 포트가 있고 포트 간에 데이터를 공유해야 할 때
- **테스트**: 한 포트가 다른 포트의 동작을 시뮬레이션하는 테스트 환경 구축
- **모니터링**: 원래 연결을 방해하지 않고 활성 포트의 데이터를 모니터링 포트에 복제
- **데이터 집계**: 여러 소스의 데이터를 여러 포트에서 전달하여 결합

## TUI에서의 구성

### 1단계: 소스 포트가 실행 중인지 확인

포트 포워딩을 구성하기 전에 소스 포트가 이미 구성되어 실행 중인지 확인합니다:

1. Entry 페이지에서 소스 포트로 이동합니다
2. Modbus 국번을 구성합니다 (마스터 또는 슬레이브 모드)
3. `Ctrl+S`로 구성을 저장하여 포트를 활성화합니다
4. 포트에 "Running ●" 상태가 표시되는지 확인합니다

### 2단계: 포트 포워딩으로 대상 포트 구성

1. 대상 포트(데이터를 전달할 포트)로 이동합니다
2. `Enter`를 눌러 Config Panel에 들어갑니다
3. 아래로 이동하여 "Enter Business Configuration"에서 `Enter`를 누릅니다
4. "Data Source" 필드로 이동합니다
5. `Enter`를 눌러 데이터 소스를 편집합니다
6. 화살표 키(`←` / `→`)를 사용하여 "Port Forwarding"이 나올 때까지 옵션을 순환합니다
7. `Enter`를 눌러 확인합니다

### 3단계: 소스 포트 선택

"Port Forwarding"을 데이터 소스로 선택한 후:

1. "Source Port" 필드로 이동합니다
2. `Enter`를 눌러 포트 선택기를 엽니다
3. 화살표 키(`←` / `→`)를 사용하여 사용 가능한 포트를 탐색합니다
4. `Enter`를 눌러 원하는 소스 포트를 선택합니다
5. `Ctrl+S`를 눌러 저장하고 포워딩을 활성화합니다

**참고**: 포트가 하나만 있는 경우(현재 포트), "Source Port" 필드에 회색으로 "No other ports available"이라고 표시되며 `Enter`를 눌러도 아무 일도 일어나지 않습니다.

### 4단계: 국번 구성

포트 포워딩이 활성화된 상태에서도 대상 포트에 최소 하나의 국번을 구성해야 합니다:

1. "Create Station"으로 이동합니다
2. Station ID, Register Type, Start Address 및 Register Count를 구성합니다
3. 레지스터 값은 소스 포트의 데이터에서 자동으로 채워집니다

### 5단계: 저장 및 활성화

1. `Ctrl+S`를 눌러 구성을 저장합니다
2. 포트가 "Running ●" 상태로 실행을 시작합니다
3. 소스 포트의 데이터가 주기적으로 이 포트로 전달됩니다

## 작동 방식

포트 포워딩이 활성화되면:

1. **백그라운드 데몬**: TUI가 이 포트 전용 백그라운드 스레드를 생성합니다
2. **주기적 읽기**: 데몬이 소스 포트의 전역 상태에서 레지스터 값을 주기적으로 읽습니다
3. **상태 동기화**: 데몬이 내부 IPC를 통해 대상 포트의 레지스터 값을 업데이트합니다
4. **자동 업데이트**: 소스 포트의 변경 사항이 대상 포트에 자동으로 반영됩니다

포워딩은 TUI 프로세스 내에서 완전히 이루어지며, 외부 네트워크나 시리얼 통신이 필요하지 않습니다.

## 사용 사례 예제: 다중 마스터 설정

다음과 같은 상황이 있다고 가정합니다:

- `/tmp/vcom1`: 물리적 Modbus 장치에 슬레이브로 연결됨
- `/tmp/vcom2`: vcom1에서 읽는 마스터로 작동하길 원함

구성:

1. `/tmp/vcom1` 구성:
   - 모드: 슬레이브
   - Modbus 요청에 응답하도록 슬레이브 국번 구성

2. `/tmp/vcom2` 구성:
   - 모드: 마스터
   - 데이터 소스: Port Forwarding
   - 소스 포트: `/tmp/vcom1`
   - 마스터 국번 구성

결과: `/tmp/vcom2`는 마스터로 작동하지만, 데이터는 `/tmp/vcom1`의 슬레이브 응답에서 가져오므로 효과적으로 데이터를 전달합니다.

## 사용 사례 예제: 데이터 복제

다음과 같은 상황이 있다고 가정합니다:

- `/tmp/vcom1`: 외부 IPC 데이터 소스에서 읽는 메인 포트
- `/tmp/vcom2`: vcom1의 데이터를 미러링해야 하는 모니터링 포트

구성:

1. `/tmp/vcom1` 구성:
   - 모드: 마스터
   - 데이터 소스: IPC 파이프 (예: `/tmp/data_feed`)
   - 국번 구성

2. `/tmp/vcom2` 구성:
   - 모드: 마스터
   - 데이터 소스: Port Forwarding
   - 소스 포트: `/tmp/vcom1`
   - 동일한 레지스터 레이아웃으로 국번 구성

결과: 두 포트 모두 동일한 데이터를 표시하며, vcom2는 vcom1의 레지스터 값을 미러링합니다.

## 제한 사항

- **소스 포트가 실행 중이어야 함**: 포워딩이 작동하려면 소스 포트가 활성화되어 있어야 합니다
- **자기 자신으로의 포워딩 불가**: 포트는 자기 자신에서 전달할 수 없습니다
- **마스터 모드 전용**: 포트 포워딩은 마스터 국번에서만 사용할 수 있습니다
- **내부 전용**: 포워딩은 TUI 내에서 이루어지며, 외부 프로세스에서 직접 포트를 포워딩할 수 없습니다

## 문제 해결

### "No other ports available" 메시지

다음과 같은 경우 나타납니다:

- 시스템에 포트가 하나만 있는 경우 (전달할 소스 포트가 없음)
- 현재 포트가 유일한 포트인 경우
- **해결 방법**: 다른 포트를 먼저 추가하고 구성 및 활성화한 다음 포워딩을 설정하십시오

### 데이터가 업데이트되지 않는 경우

다음을 확인하십시오:

- 소스 포트가 실행 중인지 ("Running ●" 상태)
- 소스 포트에 구성된 국번이 있는지
- 대상 포트가 실행 중인지 ("Running ●" 상태)
- 두 포트가 호환되는 레지스터 유형 및 주소 범위를 사용하는지

### 옵션에 포트 포워딩이 나타나지 않는 경우

다음을 확인하십시오:

- 마스터 국번을 구성하고 있는지 (슬레이브가 아님)
- Modbus Dashboard 패널에 있는지
- "Data Source" 필드로 이동했는지

## 고급: 다중 포워딩 체인

포워딩 체인을 생성할 수 있습니다:

> Port A → Port B → Port C

단, 주의해야 합니다:

- 체인의 각 링크는 지연을 증가시킵니다
- 순환 포워딩(A → B → A)은 UI에서 방지됩니다
- 여러 포워딩 레벨을 사용하는 경우 성능을 모니터링하십시오
