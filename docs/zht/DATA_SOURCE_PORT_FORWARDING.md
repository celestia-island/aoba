# 透明轉發埠（Port Forwarding）

## 概述

透明轉發埠模式允許您在 TUI 內部將一個埠的 Modbus 資料透明地轉發到另一個埠。此功能支援以下進階用例：

1. **資料轉發**：將一個埠的從站轉換為另一個埠的主站
2. **資料複製**：將主站資料複製到多個埠進行監控或測試
3. **協議橋接**：在不同的物理埠或虛擬連線之間橋接資料

## 何時使用透明轉發埠

- **多埠場景**：當您有多個串列埠並需要在它們之間共享資料時
- **測試**：建立測試設定，其中一個埠模擬另一個埠的行為
- **監控**：將活動埠的資料複製到監控埠，而不影響原始連線
- **資料聚合**：透過從不同埠轉發來組合來自多個來源的資料

## TUI 配置

### 步驟 1：確保來源埠正在執行

在配置埠轉發之前，請確保來源埠已配置並正在執行：

1. 在入口頁面導航到來源埠
2. 配置其 Modbus 站點（主站或從站模式）
3. 使用 `Ctrl+S` 儲存配置以啟用埠
4. 驗證埠顯示"Running ●"狀態

### 步驟 2：使用透明轉發配置目標埠

1. 導航到目標埠（將轉發資料的埠）
2. 按 `Enter` 進入配置面板
3. 向下導航到"Enter Business Configuration"並按 `Enter`
4. 向下導航到"資料來源"欄位
5. 按 `Enter` 編輯資料來源
6. 使用方向鍵（`←` / `→`）循環瀏覽選項，直到到達"透明轉發埠"
7. 按 `Enter` 確認

### 步驟 3：選擇來源埠

選擇"透明轉發埠"作為資料來源後：

1. 向下導航到"資料來源埠"欄位
2. 按 `Enter` 開啟埠選擇器
3. 使用方向鍵（`←` / `→`）瀏覽可用埠
4. 按 `Enter` 選擇所需的來源埠
5. 按 `Ctrl+S` 儲存並啟用轉發

**注意**：如果只存在一個埠（當前埠），"資料來源埠"欄位將顯示灰色提示"暫無其他可用埠"，按 `Enter` 將不會執行任何操作。

### 步驟 4：配置站點

即使啟用了透明轉發，您仍然需要在目標埠上配置至少一個站點：

1. 導航到"建立站點"
2. 配置站點 ID、暫存器型別、起始位址和暫存器長度
3. 暫存器值將自動從來源埠的資料中填充

### 步驟 5：儲存並啟用

1. 按 `Ctrl+S` 儲存配置
2. 埠將開始執行，顯示"Running ●"狀態
3. 來源埠的資料將定期轉發到此埠

## 運作原理

當啟用透明轉發時：

1. **背景守護程式**：TUI 為此埠生成一個專用的背景執行緒
2. **定期讀取**：守護程式定期從來源埠的全域狀態中讀取暫存器值
3. **狀態同步**：守護程式透過內部 IPC 更新目標埠的暫存器值
4. **自動更新**：來源埠的變更會自動反映在目標埠中

轉發完全在 TUI 程序內部進行，無需外部網路或串列通訊。

## 用例範例：多主站設定

假設您有：

- `/tmp/vcom1`：作為從站連線到物理 Modbus 設備
- `/tmp/vcom2`：您希望作為主站從 vcom1 讀取

配置：

1. 配置 `/tmp/vcom1`：
   - 模式：從站
   - 配置從站以回應 Modbus 請求

2. 配置 `/tmp/vcom2`：
   - 模式：主站
   - 資料來源：透明轉發埠
   - 來源埠：`/tmp/vcom1`
   - 配置主站

結果：`/tmp/vcom2` 將作為主站執行，但其資料來自 `/tmp/vcom1` 的從站回應，有效地轉發資料。

## 用例範例：資料複製

假設您有：

- `/tmp/vcom1`：從外部 IPC 資料來源讀取的主埠
- `/tmp/vcom2`：需要映射 vcom1 資料的監控埠

配置：

1. 配置 `/tmp/vcom1`：
   - 模式：主站
   - 資料來源：IPC 管道（例如 `/tmp/data_feed`）
   - 配置站點

2. 配置 `/tmp/vcom2`：
   - 模式：主站
   - 資料來源：透明轉發埠
   - 來源埠：`/tmp/vcom1`
   - 配置具有相同暫存器佈局的站點

結果：兩個埠顯示相同的資料，vcom2 映射 vcom1 的暫存器值。

## 限制

- **來源埠必須執行**：轉發運作之前，來源埠必須啟用
- **不能自我轉發**：埠不能從自身轉發
- **僅主站模式**：透明轉發僅適用於主站
- **僅內部**：轉發在 TUI 內部進行；外部程序不能直接轉發埠

## 故障排除

### 顯示"暫無其他可用埠"訊息

出現此訊息的原因：

- 系統中只存在一個埠（沒有來源埠可以轉發）
- 當前埠是唯一的埠
- **解決方案**：首先新增另一個埠，配置並啟用它，然後設定轉發

### 資料未更新

檢查：

- 來源埠正在執行（顯示"Running ●"狀態）
- 來源埠已配置站點
- 目標埠正在執行（顯示"Running ●"狀態）
- 兩個埠使用相容的暫存器型別和位址範圍

### 選項中未顯示透明轉發

確保：

- 您正在配置主站（而不是從站）
- 您在 Modbus 儀表板面板中
- 您已導航到"資料來源"欄位

## 進階：多重轉發鏈

您可以建立轉發鏈：

> 埠 A → 埠 B → 埠 C

但是要小心：

- 鏈中的每個連結都會引入延遲
- UI 會防止循環轉發（A → B → A）
- 如果使用多個轉發層級，請監控效能
