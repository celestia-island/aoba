# ポートフォワーディング（透過的ポート転送モード）

## 概要

ポートフォワーディングモードでは、TUI 内で Modbus データをあるポートから別のポートへ透過的に転送できます。この機能により、以下のような高度なユースケースが可能になります。

1. **データ転送**: あるポートのスレーブステーションを別のポートのマスターステーションに変換する
2. **データ複製**: マスターステーションのデータを監視やテストのために複数のポートに複製する
3. **プロトコルブリッジ**: 異なる物理ポートや仮想接続間でデータをブリッジする

## ポートフォワーディングの使いどころ

- **マルチポートシナリオ**: 複数のシリアルポートがあり、それらの間でデータを共有する必要がある場合
- **テスト**: あるポートが別のポートの動作をシミュレートするテストセットアップを作成する場合
- **監視**: 元の接続を中断することなく、アクティブなポートから監視用ポートにデータを複製する場合
- **データ集約**: 異なるポートからの転送により、複数ソースのデータを集約する場合

## TUI での設定

### ステップ 1: ソースポートが実行中であることを確認

ポートフォワーディングを設定する前に、ソースポートが既に設定され実行中であることを確認してください。

1. Entry ページでソースポートに移動します
2. Modbus ステーションを設定します（マスターモードまたはスレーブモード）
3. `Ctrl+S` で設定を保存し、ポートを有効にします
4. ポートが「Running ●」ステータスであることを確認します

### ステップ 2: ターゲットポートをポートフォワーディングで設定

1. ターゲットポート（データを転送するポート）に移動します
2. `Enter` を押して設定パネルを開きます
3. 「Enter Business Configuration」に移動し、`Enter` を押します
4. 「Data Source」フィールドに移動します
5. `Enter` を押してデータソースを編集します
6. 矢印キー（`←` / `→`）で「Port Forwarding」まで切り替えます
7. `Enter` で確定します

### ステップ 3: ソースポートの選択

「Port Forwarding」をデータソースとして選択した後:

1. 「Source Port」フィールドに移動します
2. `Enter` を押してポートセレクターを開きます
3. 矢印キー（`←` / `→`）で利用可能なポート間を移動します
4. `Enter` で目的のソースポートを選択します
5. `Ctrl+S` で保存し、転送を有効にします

**注**: ポートが1つしかない（現在のポートのみ）場合、「Source Port」フィールドにはグレーアウトしたヒント「No other ports available」が表示され、`Enter` を押しても何も起こりません。

### ステップ 4: ステーションの設定

ポートフォワーディングが有効であっても、ターゲットポートに少なくとも1つのステーションを設定する必要があります。

1. 「Create Station」に移動します
2. Station ID、Register Type、Start Address、Register Count を設定します
3. レジスタ値はソースポートのデータから自動的に設定されます

### ステップ 5: 保存と有効化

1. `Ctrl+S` で設定を保存します
2. ポートが「Running ●」ステータスで実行を開始します
3. ソースポートのデータが定期的にこのポートに転送されます

## 動作の仕組み

ポートフォワーディングが有効な場合:

1. **バックグラウンドデーモン**: TUI はこのポート専用のバックグラウンドスレッドを起動します
2. **定期読み取り**: デーモンはソースポートのグローバル状態からレジスタ値を定期的に読み取ります
3. **状態同期**: デーモンは内部 IPC を介してターゲットポートのレジスタ値を更新します
4. **自動更新**: ソースポートの変更がターゲットポートに自動的に反映されます

転送はすべて TUI プロセス内で行われ、外部のネットワークやシリアル通信は必要ありません。

## 使用例: マルチマスター構成

以下のような構成を考えます。

- `/tmp/vcom1`: 物理 Modbus デバイスにスレーブとして接続
- `/tmp/vcom2`: vcom1 から読み取るマスターとして動作させたい

設定:

1. `/tmp/vcom1` の設定:
   - モード: Slave
   - Modbus リクエストに応答するスレーブステーションを設定

2. `/tmp/vcom2` の設定:
   - モード: Master
   - Data Source: Port Forwarding
   - Source Port: `/tmp/vcom1`
   - マスターステーションを設定

結果: `/tmp/vcom2` はマスターとして動作しますが、データは `/tmp/vcom1` のスレーブレスポンスから取得され、実質的にデータが転送されます。

## 使用例: データ複製

以下のような構成を考えます。

- `/tmp/vcom1`: 外部 IPC データソースから読み取るメインポート
- `/tmp/vcom2`: vcom1 のデータをミラーする必要がある監視ポート

設定:

1. `/tmp/vcom1` の設定:
   - モード: Master
   - Data Source: IPC Pipe（例: `/tmp/data_feed`）
   - ステーションを設定

2. `/tmp/vcom2` の設定:
   - モード: Master
   - Data Source: Port Forwarding
   - Source Port: `/tmp/vcom1`
   - 同じレジスタレイアウトでステーションを設定

結果: 両方のポートが同じデータを表示し、vcom2 が vcom1 のレジスタ値をミラーします。

## 制限事項

- **ソースポートは実行中である必要があります**: 転送が動作する前にソースポートが有効になっている必要があります
- **自己転送は不可**: ポートは自分自身から転送できません
- **マスターモードのみ**: ポートフォワーディングはマスターステーションのみで利用可能です
- **内部のみ**: 転送は TUI 内で行われます。外部プロセスは直接ポートを転送できません

## トラブルシューティング

### 「No other ports available」メッセージ

以下の場合に表示されます。

- システムにポートが1つしか存在しない（転送元のソースポートがない）
- 現在のポートが唯一のポートである
- **解決策**: まず別のポートを追加し、設定・有効化してから転送を設定してください

### データが更新されない

以下を確認してください。

- ソースポートが実行中である（「Running ●」ステータス）
- ソースポートにステーションが設定されている
- ターゲットポートが実行中である（「Running ●」ステータス）
- 両方のポートで互換性のあるレジスタタイプとアドレス範囲が使用されている

### ポートフォワーディングがオプションに表示されない

以下を確認してください。

- マスターステーションを設定している（スレーブではない）
- Modbus Dashboard パネルにいる
- 「Data Source」フィールドに移動している

## 応用: 複数の転送チェーン

転送チェーンを作成できます。

> ポート A → ポート B → ポート C

ただし、以下に注意してください。

- チェーンの各リンクはレイテンシを増加させます
- 循環転送（A → B → A）は UI で防止されます
- 複数レベルの転送を使用する場合はパフォーマンスを監視してください
