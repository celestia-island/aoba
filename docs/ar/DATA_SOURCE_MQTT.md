# مصدر بيانات مخصص — MQTT

## بداية سريعة — تشغيل مستقبل CLI بسيط

ابدأ تشغيل تطبيق CLI بحيث يشترك في موضوع MQTT ويعمل كمستقبل. مثال (يُشغّل من جذر المستودع):

```bash
# باستخدام cargo (موصى به أثناء التطوير)
cargo run --bin aoba -- --master-provide-persist /tmp/vcom1 \
  --register-mode holding --register-address 0 --register-length 10 \
  --data-source mqtt://localhost:1883/aoba/data/in

# أو، إذا قمت ببناء الملف الثنائي:
./target/debug/aoba --master-provide-persist /tmp/vcom1 --data-source mqtt://localhost:1883/aoba/data/in
```

يتضمن رابط `mqtt://.../<topic>` مسار الموضوع (مثل `aoba/data/in`) وسيشترك CLI في ذلك الموضوع.

## نظرة عامة

يصف هذا المستند كيفية نشر الرسائل إلى مصدر البيانات المخصص القائم على MQTT في التطبيق. يتضمن تكوين الوسيط/الاتصال، وأسماء الموضوعات الموصى بها، ومثال حمولة `mosquitto_pub` لتنفيذ إرسال بيانات downwards.

## الوسيط / الاتصال

- المضيف: `mqtt.example.com` أو `localhost`
- المنفذ: `1883` (نص عادي) أو `8883` (TLS)
- اسم المستخدم/كلمة المرور: اختياري — إذا كان وسيطك يتطلب مصادقة، قدمها في تكوين العميل
- TLS: إذا كنت تستخدم `8883`، قدم شهادة CA وشهادة/مفتاح العميل حيثما يلزم

## الموضوعات الموصى بها

- الوارد (إلى التطبيق): `aoba/data/in` — يشترك التطبيق هنا لاستقبال البيانات أو الأوامر الصاعدة
- الصادر (إلى الجهاز/vcom): `aoba/data/out/<port>` — ينشر التطبيق رسائل صادرة معالجة موجهة إلى منفذ معين (مثال: `aoba/data/out/tmp_vcom1`)

## تنسيق الحمولة

يتوقع التطبيق حمولات JSON. المخطط الدقيق مرن لكن المثال التالي هو شكل عملي لكل من تحديثات الحالة وأوامر الإرسال downwards:

```json
{
  "source": "mqtt",
  "timestamp": "2025-11-15T12:34:56Z",
  "port": "/tmp/vcom1",
  "type": "downlink",
  "body": {
    "command": "write_register",
    "registers": [{"address":0, "value": "1234"}]
  }
}
```

## مثال: نشر إرسال downwards باستخدام mosquitto_pub

ينشر هذا المثال إرسالاً downwards إلى الموضوع الوارد الذي سيعالجه التطبيق ثم ينفذ الكتابة الفعلية إلى `port` المُكوّن.

```bash
mosquitto_pub -h localhost -p 1883 -t "aoba/data/in" -u "user" -P "pass" -m '{
  "source":"mqtt",
  "timestamp":"2025-11-15T12:34:56Z",
  "port":"/tmp/vcom1",
  "type":"downlink",
  "body":{ "command":"write_register", "registers":[{"address":0,"value":"1234"}] }
}'
```

## ملاحظات ونصائح

- استخدم أسماء موضوعات قابلة للتوقع لتبسيط التصفية والصلاحيات.
- عند استهداف مسار منفذ تسلسلي فعلي (مثل `/tmp/vcom1`) تجنب الأحرف التي قد تسبب مشكلات في تحليل الموضوع؛ يمكنك تعيين أسماء المنافذ إلى تسميات آمنة للموضوع في التكوين.
- إذا كان وسيطك يدعم الرسائل المحتفظة، كن حذرًا: يمكن إعادة تطبيق رسائل الإرسال downwards المحتفظة عند إعادة الاتصال.

إذا كنت تريد نموذج تكوين وسيط أو أداة اختبار آلية (مثل نص برمجي صغير ينشر سلسلة من الإرسالات downwards وينتظر تأكيد حالة CLI/TUI)، أخبرني بأدواتك المفضلة ويمكنني إضافتها.
