# مصدر بيانات مخصص — HTTP

## بداية سريعة — تشغيل مستقبل CLI بسيط

ابدأ تشغيل تطبيق CLI في وضع يعمل كمستقبل (سيستضيف CLI نقطة نهاية HTTP ويطبق JSON المُرسل). مثال (يُشغّل من جذر المستودع):

```bash
# باستخدام cargo (موصى به أثناء التطوير)
cargo run --bin aoba -- --master-provide-persist /tmp/vcom1 \
  --register-mode holding --register-address 0 --register-length 10 \
  --data-source http://8080

# أو، إذا قمت ببناء الملف الثنائي:
./target/debug/aoba --master-provide-persist /tmp/vcom1 --data-source http://8080
```

يقوم الأمر أعلاه بتشغيل خادم HTTP مرتبط بـ `127.0.0.1:8080` ويقبل طلبات `POST` إلى `/` (الجذر). استخدم مثال `curl` أدناه لإرسال بيانات POST.

## نظرة عامة

يصف هذا المستند مصدر البيانات المخصص HTTP الذي يستخدمه التطبيق. يعرض شكل الطلب المتوقع، والترويسات الشائعة، ومثال `curl` بسيط يمكنك استخدامه للتحقق السريع من التكامل.

## نقطة النهاية

- الطريقة: `POST`
- الرابط: `http://<host>:<port>/` (مثال: `http://localhost:8080/`)
- نوع المحتوى: `application/json`

## تنسيق الطلب

تقبل الخدمة جسم JSON. يبدو مثال الحمولة البسيط كالتالي:

```json
{
  "source": "http",
  "timestamp": "2025-11-15T12:34:56Z",
  "port": "/tmp/vcom1",
  "payload": {
    "type": "register_update",
    "registers": [
      {"address": 0, "value": "1234"},
      {"address": 1, "value": "abcd"}
    ]
  }
}
```

ملاحظات:

- استخدم ISO 8601 لـ `timestamp` عند توفره.
- محتوى `payload` خاص بالتطبيق؛ يُظهر المثال أعلاه تحديثًا شائعًا بنمط السجلات.

## مثال اختبار curl

استبدل `<host>` و `<port>` بالخادم العامل لديك. يرسل أمر `curl` هذا حمولة JSON أعلاه:

```bash
curl -v -X POST "http://localhost:8080/" \
  -H "Content-Type: application/json" \
  -d '{
    "source":"http",
    "timestamp":"2025-11-15T12:34:56Z",
    "port":"/tmp/vcom1",
    "payload":{
      "type":"register_update",
      "registers":[{"address":0,"value":"1234"}]
    }
  }'
```

## السلوك المتوقع

- `200 OK` (أو `202 Accepted`) للرسائل المقبولة/الموضوعة في الطابور.
- إذا أعاد الخادم خطأً (4xx/5xx)، افحص جسم الاستجابة للتفاصيل.

## نصائح واستكشاف الأخطاء

- تأكد من وجود ترويسة `Content-Type: application/json`.
- إذا كان خادمك يتطلب مصادقة، أضف ترويسة `Authorization` المناسبة (مثل `Bearer <token>`).
- للحمولات الكبيرة، فكّر في الاختبار باستخدام `--data-binary` وزيادة مهل الخادم.
