# Перенаправление портов (режим прозрачной пересылки портов)

## Обзор

Режим перенаправления портов позволяет прозрачно пересылать данные Modbus с одного порта на другой в пределах TUI. Эта функция поддерживает такие расширенные сценарии:

1. **Пересылка данных**: Преобразование станции slave с одного порта в станцию master на другом порту
2. **Репликация данных**: Дублирование данных станции master на несколько портов для мониторинга или тестирования
3. **Мост протоколов**: Обмен данными между различными физическими портами или виртуальными соединениями

## Когда использовать перенаправление портов

- **Многопортовые сценарии**: Когда имеется несколько последовательных портов и необходимо обмениваться данными между ними
- **Тестирование**: Создание тестовых конфигураций, где один порт имитирует поведение другого
- **Мониторинг**: Репликация данных с активного порта на порт мониторинга без нарушения исходного соединения
- **Агрегация данных**: Объединение данных из нескольких источников путём пересылки с различных портов

## Настройка в TUI

### Шаг 1: Убедитесь, что исходный порт запущен

Перед настройкой перенаправления портов убедитесь, что исходный порт уже настроен и запущен:

1. Перейдите к исходному порту на странице Entry
2. Настройте его станции Modbus (режим master или slave)
3. Сохраните конфигурацию с помощью `Ctrl+S` для активации порта
4. Убедитесь, что порт отображает статус "Running ●"

### Шаг 2: Настройка целевого порта с перенаправлением

1. Перейдите к целевому порту (порту, который будет пересылать данные)
2. Нажмите `Enter` для входа в панель конфигурации
3. Перейдите вниз к пункту "Enter Business Configuration" и нажмите `Enter`
4. Перейдите вниз к полю "Data Source"
5. Нажмите `Enter` для редактирования источника данных
6. Используйте клавиши со стрелками (`←` / `→`) для переключения между параметрами, пока не будет выбрано "Port Forwarding"
7. Нажмите `Enter` для подтверждения

### Шаг 3: Выбор исходного порта

После выбора "Port Forwarding" в качестве источника данных:

1. Перейдите вниз к полю "Source Port"
2. Нажмите `Enter` для открытия выбора порта
3. Используйте клавиши со стрелками (`←` / `→`) для навигации по доступным портам
4. Нажмите `Enter` для выбора нужного исходного порта
5. Нажмите `Ctrl+S` для сохранения и активации перенаправления

**Примечание**: Если существует только один порт (текущий порт), поле "Source Port" отобразит серую подсказку "No other ports available", и нажатие `Enter` не приведёт к действию.

### Шаг 4: Настройка станции

Даже при включённом перенаправлении портов необходимо настроить хотя бы одну станцию на целевом порту:

1. Перейдите к "Create Station"
2. Настройте Station ID, Register Type, Start Address и Register Count
3. Значения регистров будут автоматически заполнены из данных исходного порта

### Шаг 5: Сохранение и активация

1. Нажмите `Ctrl+S` для сохранения конфигурации
2. Порт начнёт работу со статусом "Running ●"
3. Данные из исходного порта будут периодически пересылаться на этот порт

## Как это работает

При включённом перенаправлении портов:

1. **Фоновый демон**: TUI создаёт фоновый поток, выделенный для этого порта
2. **Периодическое чтение**: Демон периодически считывает значения регистров из глобального состояния исходного порта
3. **Синхронизация состояния**: Демон обновляет значения регистров целевого порта через внутренний IPC
4. **Автоматические обновления**: Изменения на исходном порту автоматически отражаются на целевом порту

Пересылка происходит полностью внутри процесса TUI, без необходимости внешнего сетевого или последовательного обмена.

## Пример сценария: конфигурация с несколькими master

Предположим:

- `/tmp/vcom1`: Подключён к физическому устройству Modbus в режиме slave
- `/tmp/vcom2`: Должен работать в режиме master, считывая данные с vcom1

Конфигурация:

1. Настройка `/tmp/vcom1`:
   - Режим: Slave
   - Настройте станции slave для ответа на запросы Modbus

2. Настройка `/tmp/vcom2`:
   - Режим: Master
   - Data Source: Port Forwarding
   - Source Port: `/tmp/vcom1`
   - Настройте станции master

Результат: `/tmp/vcom2` будет работать как master, но его данные будут поступать из ответов slave порта `/tmp/vcom1`, фактически пересылая данные.

## Пример сценария: репликация данных

Предположим:

- `/tmp/vcom1`: Основной порт, считывающий данные из внешнего IPC-источника
- `/tmp/vcom2`: Порт мониторинга, который должен зеркалировать данные vcom1

Конфигурация:

1. Настройка `/tmp/vcom1`:
   - Режим: Master
   - Data Source: IPC Pipe (например, `/tmp/data_feed`)
   - Настройте станции

2. Настройка `/tmp/vcom2`:
   - Режим: Master
   - Data Source: Port Forwarding
   - Source Port: `/tmp/vcom1`
   - Настройте станции с такой же структурой регистров

Результат: Оба порта отображают одинаковые данные, при этом vcom2 зеркалирует значения регистров vcom1.

## Ограничения

- **Исходный порт должен быть запущен**: Исходный порт должен быть активирован до начала пересылки
- **Без самомаршрутизации**: Порт не может пересылать данные сам себе
- **Только режим Master**: Перенаправление портов доступно только для станций master
- **Только внутри TUI**: Пересылка происходит внутри TUI; внешние процессы не могут напрямую перенаправлять порты

## Устранение неполадок

### Сообщение "No other ports available"

Это сообщение появляется, когда:

- В системе существует только один порт (нет исходного порта для пересылки)
- Текущий порт — единственный порт
- **Решение**: Сначала добавьте другой порт, настройте и активируйте его, затем настройте пересылку

### Данные не обновляются

Проверьте:

- Исходный порт запущен (отображает статус "Running ●")
- На исходном порту настроены станции
- Целевой порт запущен (отображает статус "Running ●")
- Оба порта используют совместимые типы регистров и диапазоны адресов

### Перенаправление портов не появляется в параметрах

Убедитесь:

- Вы настраиваете станцию master (не slave)
- Вы находитесь на панели Modbus Dashboard
- Вы перешли к полю "Data Source"

## Дополнительно: множественные цепочки перенаправления

Можно создавать цепочки перенаправления:

> Порт A → Порт B → Порт C

Однако будьте осторожны:

- Каждое звено в цепочке добавляет задержку
- Циклическое перенаправление (A → B → A) предотвращается интерфейсом
- Контролируйте производительность при использовании нескольких уровней перенаправления
